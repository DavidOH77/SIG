# SIG 아키텍처 노트

## 현재 구현(하이브리드 MVP)
1. **Python 분석 서버 (`server.py`)**
   - 데이터 업로드/파싱/스키마추론
   - 데이터 건강도, 정량, 세그먼트, 텍스트, 우선순위 분석
   - 리포트 생성 및 CSV/Markdown export
2. **Next.js App Router 셸 (`app/*`)**
   - 홈/프로젝트 진입 화면
   - Python 분석 UI를 iframe으로 연결(점진적 전환)

## 왜 이렇게 했는가
- 먼저 분석 기능을 빠르게 검증하기 위해 Python 단일 서버로 MVP를 완성
- 이후 Next.js를 추가해 프론트엔드 계층을 분리하는 방향으로 확장

## 단계적 전환 계획
- 1단계: Next 셸 + Python 페이지 임베드 (현재)
- 2단계: Next Route Handler/API로 Python JSON 데이터 호출
- 3단계: Next 네이티브 차트/컴포넌트로 완전 전환

## 분석 원칙
- 모든 인사이트는 계산 근거(통계/클러스터/인용문)와 함께 제시
- 드라이버 분석은 인과가 아닌 **연관 요인**으로 표기
